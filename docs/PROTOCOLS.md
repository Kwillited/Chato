# 协议与集成文档

## 1. 概述

本文档详细描述了 Chato 应用使用的各种协议和集成方式，包括 API 协议、通信协议、第三方服务集成等。

## 2. API 协议

### 2.1 RESTful API

#### 2.1.1 基本规范

- **URL 设计**: 采用 RESTful 风格，使用名词表示资源，动词表示操作
- **HTTP 方法**: GET（获取）、POST（创建）、PUT（更新）、DELETE（删除）
- **状态码**: 使用标准 HTTP 状态码
- **请求格式**: JSON
- **响应格式**: JSON

#### 2.1.2 认证机制

- **API Key**: 使用 Bearer Token 认证
- **认证头**: `Authorization: Bearer <api_key>`
- **权限管理**: 基于角色的访问控制（RBAC）

#### 2.1.3 错误处理

- 使用标准 HTTP 状态码
- 统一的错误响应格式
- 详细的错误信息

### 2.2 WebSocket API

#### 2.2.1 基本规范

- **URL 格式**: `ws://<hostname>:<port>/ws/<endpoint>`
- **消息格式**: JSON
- **连接管理**: 支持心跳检测

#### 2.2.2 消息类型

| 类型 | 描述 |
|------|------|
| message | 聊天消息 |
| stream | 流式响应 |
| error | 错误信息 |
| ping | 心跳检测 |
| pong | 心跳响应 |

#### 2.2.3 示例

```json
// 客户端发送
{
  "type": "message",
  "data": {
    "content": "你好",
    "role": "user"
  }
}

// 服务器响应
{
  "type": "stream",
  "data": {
    "content": "正在生成...",
    "done": false
  }
}

{
  "type": "stream",
  "data": {
    "content": "你好，我是 AI 助手",
    "done": true
  }
}
```

## 3. 通信协议

### 3.1 前后端通信

#### 3.1.1 Tauri IPC

- **机制**: 使用 Tauri 提供的 IPC（进程间通信）
- **前端调用**: `window.__TAURI__.invoke('<command>', <args>)`
- **后端调用**: `tauri::event::emit()`
- **优点**: 安全、高效、跨平台

#### 3.1.2 HTTP 通信

- **机制**: 使用 fetch API 或 axios 进行 HTTP 请求
- **基础 URL**: 开发环境下为 `http://localhost:8000/api/v1`
- **支持**: RESTful API 和 WebSocket

### 3.2 后端服务通信

#### 3.2.1 Python 内部通信

- **机制**: 函数调用、异步通信
- **框架**: FastAPI 异步框架
- **依赖注入**: 使用依赖注入管理服务

#### 3.2.2 Rust 与 Python 通信

- **机制**: Tauri 命令调用、子进程通信
- **Rust 调用 Python**: 通过子进程或 FFI
- **Python 调用 Rust**: 通过 Tauri 事件

## 11. 集成开发指南

### 11.1 集成新的 AI 模型

1. **创建模型适配器**: 继承 `base_model.py`，实现必要的方法
2. **注册模型**: 在 `model_manager.py` 中注册新模型
3. **添加 API 路由**: 在 `models_router.py` 中添加相关路由
4. **更新前端**: 添加模型选择选项

### 11.2 集成新的文档格式

1. **添加文档加载器**: 在 `document_loader.py` 中添加新的文档加载器
2. **实现解析逻辑**: 实现文档解析和文本提取
3. **更新文件类型列表**: 在前端和后端更新支持的文件类型

### 11.3 集成新的向量数据库

1. **添加向量数据库服务**: 在 `vector_db_service.py` 中添加新的数据库服务
2. **实现向量操作**: 实现向量的存储、检索、更新和删除
3. **更新配置**: 在配置文件中添加新数据库的配置选项

## 13. 故障排除

### 13.1 常见集成问题

| 问题 | 可能原因 | 解决方案 |
|------|----------|----------|
| API 调用失败 | 网络问题、认证错误 | 检查网络连接、验证 API Key |
| 模型调用超时 | 模型响应慢、网络问题 | 调整超时设置、检查网络 |
| 文档解析失败 | 不支持的格式、文件损坏 | 检查文件格式、修复损坏文件 |
| 向量检索不准确 | 嵌入模型问题、参数设置 | 调整嵌入模型、优化检索参数 |
